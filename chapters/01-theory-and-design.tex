\section{Выводы по первой главе}

В данной главе сформирован всесторонний теоретический и архитектурный базис для разработки системы динамической маршрутизации. Анализ существующих технологий показал, что классические системы, работающие в оперативной памяти (OSRM, Valhalla), несмотря на высокую скорость поиска, обладают критическим недостатком для мультиагентных систем — невозможностью мгновенного обновления весовых коэффициентов графа без перестроения индексов. На основании этого обоснован выбор СУБД-ориентированной архитектуры на базе СУБД PostgreSQL и расширения pgRouting, гарантирующих транзакционную целостность и работу с актуальным состоянием графа через механизм многоверсионного управления доступом (MVCC).

Для моделирования транспортных потоков введена математическая зависимость, базирующаяся на модели Гриншилдса, которая связывает эффективную скорость движения с плотностью трафика, позволяя динамически корректировать веса ребер в реальном времени.

С целью преодоления ограничений производительности, свойственных SQL-решениям, разработан комплекс алгоритмических оптимизаций:
\begin{itemize}
    \item \textbf{метод пространственного секционирования} для сегментации графа, решающий проблему исчерпания памяти при построении топологии;
    \item \textbf{алгоритм поиска сильно связных компонент (алгоритм Тарьяна)} для выявления и исключения из маршрутизации топологически изолированных подграфов (<<островов>>);
    \item \textbf{динамическая область поиска}, отсекающая до 90\% нерелевантного пространства поиска при выполнении алгоритма Дейкстры;
    \item \textbf{метод итеративных штрафов} для нахождения топологически независимых альтернативных маршрутов без блокировки графа.
\end{itemize}

Сформулированные теоретические положения и математические модели создают необходимый фундамент для программной реализации подсистем, которая будет детально рассмотрена в следующей главе.
